Thiết kế Hệ thống Đấu giá bằng Java OOP: Phân tích kiến trúc Class Auction và Entity

06 Tháng 6, 2026
Trần Quốc Khánh
Software Architecture

Một trong những lỗi thiết kế kinh điển mà các mô hình ngôn ngữ lớn (LLMs) thường xuyên mắc phải là lạm dụng tính kế thừa (Inheritance) nhằm mục đích tái sử dụng mã nguồn một cách cơ học. Bài viết này phân tích kiến trúc lập trình hệ thống đấu giá, qua đó chứng minh tại sao quan hệ chứa đựng (Composition) lại tối ưu hơn kế thừa trong trường hợp này.

// AI đề xuất: Class Auction kế thừa trực tiếp Entity public class Auction extends Entity { private String title; private BigDecimal currentBid; public Auction(String id, LocalDateTime createdAt, String title) { super(id, createdAt); // Ép buộc liên kết this.title = title; } }
// Thiết kế chuẩn: Tách biệt Entity lưu trữ khỏi nghiệp vụ public class Auction { private final AuctionEntity data; // Composition (HAS-A relation) private final List<Bid> bidHistory; public Auction(AuctionEntity data) { this.data = data; this.bidHistory = new ArrayList<>(); } public void placeBid(Bid newBid) { if (data.isExpired()) throw new IllegalStateException("Đã kết thúc!"); bidHistory.add(newBid); } }
Vấn đề thiết kế của AI: Kế thừa đại diện cho quan hệ "IS-A" (Là một). Một cuộc đấu giá không phải là một thực thể lưu trữ dữ liệu thô (Entity) đơn thuần; nó đại diện cho một quy trình nghiệp vụ (Domain Process). Việc ép kế thừa trực tiếp gây ra sự kết chặt và rò rỉ dữ liệu.
Phản biện và Sửa đổi cá nhân (>50%): Việc sử dụng Composition tách biệt hoàn toàn cơ chế lưu trữ dữ liệu (Data Persistence) ra khỏi quy trình kiểm soát nghiệp vụ (Domain Logic), giúp lớp nghiệp vụ cực kỳ linh hoạt, giảm thiểu kết chặt và dễ dàng thực hiện Unit Test.

An Toàn Học Thuật & Sử Dụng AI Có Trách Nhiệm

Chính sách VNU & Rào cản kỹ thuật

VNU đã chủ động đưa AI trở thành học phần bắt buộc đối với sinh viên từ năm 2025. Đây là chính sách cởi mở, thúc đẩy năng suất. Tuy nhiên, thách thức đối với sinh viên CNTT là rất lớn. Việc phụ thuộc hoàn toàn vào AI để giải quyết các bài toán giải thuật cốt lõi sẽ tạo ra **"ảo tưởng năng lực"**, thui chột khả năng tư duy gỡ lỗi độc lập khi đi làm thực tế.
